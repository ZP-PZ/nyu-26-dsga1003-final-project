# Reproducibility Guide

This document describes how to reproduce the project pipeline and how to inspect the saved results included in the repository.

## Environment

Recommended setup:

```bash
conda env create -f environment.yml
conda activate residual-reaggregation
```

Alternative pip setup:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

The project expects Python 3.11. Training and evaluation use PyTorch, Hugging Face Transformers, Hugging Face Datasets, PyArrow, and tqdm. Figure generation uses matplotlib.

## Hardware

The scripts are written to run on CPU, but full training and evaluation of `Qwen/Qwen3-0.6B` are intended for a CUDA GPU. The implemented defaults were chosen for a laptop GPU with about 12GB VRAM:

- mixed precision on CUDA
- SDPA attention backend
- TF32 enabled on CUDA
- micro-batches plus gradient accumulation

Running the full Stage 2 training on CPU is not recommended.

## Included and excluded artifacts

Included:

```text
artifact/data/raw/
artifact/data/prepared/
result/metrics/
result/analysis/
result/tables/
result/figures/
script/
```

Excluded:

```text
artifact/model/
artifact/checkpoints/
*.pt
*.pth
*.bin
*.safetensors
```

The excluded artifacts are large local model/checkpoint files. They can be regenerated by running Stage 0 and Stage 2.

## Stage 0: download model and raw data

```bash
python script/0_download_model_and_data.py
```

Expected outputs:

```text
artifact/model/qwen3_0_6b/
artifact/data/raw/wikitext_2_raw/
artifact/data/raw/medical_abstracts/
artifact/data/raw/download_manifest.json
```

Default sources:

- model: `Qwen/Qwen3-0.6B`
- general corpus: `wikitext`, config `wikitext-2-raw-v1`
- specialized corpus: `TimSchopf/medical_abstracts`

## Stage 1: prepare continuation data

```bash
python script/1_prepare_data.py
```

Expected outputs:

```text
artifact/data/prepared/train.arrow
artifact/data/prepared/validation.arrow
artifact/data/prepared/test.arrow
artifact/data/prepared/summary.json
```

Default preprocessing:

- seed: `42`
- train/validation/test split: `0.8/0.1/0.1`
- prompt length: `96` tokens
- target length: `96` tokens
- window step: `192` tokens
- loss labels ignore prompt tokens with `-100`

## Stage 2: train lightweight controllers

```bash
python script/2_train_static.py
python script/2_train_prompt_conditioned.py
python script/2_train_static_reaggregation.py
python script/2_train_prompt_conditioned_reaggregation.py
```

Expected outputs:

```text
artifact/checkpoints/static_write_strength/
artifact/checkpoints/prompt_conditioned_write_strength/
artifact/checkpoints/static_residual_stream_reaggregation/
artifact/checkpoints/prompt_conditioned_residual_stream_reaggregation/
```

Each checkpoint directory contains:

```text
trained_model.pt
train_config.json
train_loss_history.json
train_epoch_history.json
```

The checkpoint directories are ignored by Git because they contain binary artifacts.

## Stage 3: evaluate models

```bash
python script/3_eval_frozen_base.py
python script/3_eval_static.py
python script/3_eval_prompt_conditioned.py
python script/3_eval_static_reaggregation.py
python script/3_eval_prompt_conditioned_reaggregation.py
```

Expected outputs:

```text
result/metrics/frozen_base.json
result/metrics/static_write_strength.json
result/metrics/prompt_conditioned_write_strength.json
result/metrics/static_residual_stream_reaggregation.json
result/metrics/prompt_conditioned_residual_stream_reaggregation.json
```

The included repository results use older filenames for write-strength variants:

```text
result/metrics/static_residual.json
result/metrics/prompt_conditioned.json
```

The plotting script supports both naming conventions.

## Stage 4: inspect prompt-conditioned controllers

```bash
python script/4_inspect_MLPs.py
```

Expected output:

```text
result/analysis/mlp_inspection.json
```

The inspection script reports:

- average pairwise collapse cosine
- domain-separation cosine gap
- per-layer raw-output variance
- per-layer domain-mean difference

## Stage 5: regenerate tables and figures

```bash
python script/5_make_plots.py
```

Expected outputs:

```text
result/tables/metrics_summary.csv
result/figures/01_test_perplexity_comparison.{png,pdf}
result/figures/02_test_perplexity_by_source.{png,pdf}
result/figures/03_relative_improvement_over_frozen_base.{png,pdf}
result/figures/04_mlp_inspection_summary.{png,pdf}
result/figures/05_per_layer_mlp_behavior.{png,pdf}
```

Stage 5 only reads saved JSON files and does not require the Qwen model or checkpoints.

## Lightweight verification

To verify the repository without rerunning training:

```bash
python -m py_compile script/*.py
python script/5_make_plots.py
```

This checks Python syntax and regenerates paper figures from saved metrics.

## Known caveats

- Full reproduction requires network access to Hugging Face for Stage 0.
- Exact training metrics may vary slightly across hardware, PyTorch versions, and mixed-precision settings.
- The saved results are from a single run per model variant and do not include error bars.
- The repository includes processed data for easy inspection, but not trained checkpoint binaries.
